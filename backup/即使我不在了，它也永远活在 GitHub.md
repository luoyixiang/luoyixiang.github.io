# 即使我不在了，它也永远活在 GitHub

> “When I'm six feet under, this will still be running on GitHub's servers rent-free.”

前几天改博客配置的时候，我突然想到一件事：

**这个博客，可能比我活得久。**

## 它不在我电脑上

传统博客搭在本地、配数据库、装 WordPress、买服务器、续费域名……  
哪一天我不想管了，它就真的没了。

但这个 Gmeek 不一样。

它没有数据库，没有后端，没有服务器要维护。  
它只是一堆 Markdown 文件 + GitHub Actions 流水线 + GitHub Pages 托管。

**全部家当，都在 GitHub 上。**

## 我不在了，它还在

GitHub 仓库不会因为我哪天不登录就消失。  
Actions 构建也不会因为我忘了续费就停掉。

只要 GitHub 还活着（以微软的财力，应该还能撑很久），  
这个博客就会一直在。

- Issues 里写的文章，永远在
- 评论区的讨论，永远在
- 那些随手记下的笔记、吐槽、技术踩坑，**全都在**

它甚至不需要我。

## 一种奇怪的“永生”

我其实不是想说什么沉重的话题。

只是觉得这件事挺有意思的：

> 一个人写的字，放在一个自己管不了的平台上，  
> 反而比自己管更“永垂不朽”。

我没法保证十年后还有精力折腾博客。  
但我几乎可以确定：十年后 `https://luoyixiang.github.io` 还能打开。

如果打不开，那一定是 GitHub 出问题了——  
而那时候，全世界的程序员应该都在跟我一起骂。

## 写在最后

这句话是我自己调侃自己用的：

**“I'm mortal. This repo lives rent-free on GitHub forever.”**

我不是什么大人物，写的也不是什么传世经典。  
但这个博客会像一个安静的幽灵，一直跑在 GitHub 的服务器上。

不占资源，不要钱，也不需要我。

**白嫖到地底下。**

---

*—— 写于配置好 Gmeek 的那个清晨*
